\section{Problem statement}

The goal of this work is to forecast the values of a time series using the econometric models covered in the course.

The ARIMA(p, d, q) model extends ARMA-type models to non-stationary time series that can be made stationary after differencing. The ARIMA(p, d, q) model for a series $y_t$ is defined as an ARMA(p, q) model applied to the $d$-th order differences of $y_t$.

The $\textbf{ARIMA(p, d, q)}$ model:
\begin{equation*}
	\Delta^d y_t = \alpha_1 \Delta^d y_{t-1} + ... + \alpha_p\Delta^dy_{t-p} + \varepsilon_t + \beta_1\varepsilon_{t-1} + ... + \beta_q\varepsilon_{t-q},
\end{equation*}

where $y_t$ is the value of the time series at time $t$, $\Delta^d = (1 - L)^d$, $\varepsilon_t$ is white noise, and $L$ is the lag operator.

The generalization of the ARIMA model to series with a seasonal component is called SARIMA. Let $s$ be the known seasonality of the series. We add to the ARIMA(p, d, q) model the components responsible for values in previous seasons. The SARIMA model can then be written as follows:

The $\textbf{SARIMA(p, d, q)(P, D, Q)s}$ model:

\begin{equation*}
	\begin{split}
		\Delta^D_s\Delta^d y_t
		&= \alpha_1\Delta^D_s \Delta^d y_{t-1} + \dots + \alpha_p\Delta^D_s\Delta^dy_{t-p} \\
		&\quad+ \varepsilon_t + \beta_1\varepsilon_{t-1} + \dots + \beta_q\varepsilon_{t-q} \\
		&\quad+ \alpha_1^s\Delta^D_s \Delta^d y_{t-s} + \dots + \alpha_p^s\Delta^D_s \Delta^d y_{t-ps} \\
		&\quad+ \beta_1^s\varepsilon_{t-s}+\beta_Q^s\varepsilon_{t-Qs}.
	\end{split}
\end{equation*}

where $\Delta^D_s = (1 - L^s)^D$.



A further generalization of SARIMA is the SARIMAX model. SARIMAX (Seasonal AutoRegressive Integrated Moving Average with eXogenous inputs) is an econometric model for time-series forecasting that accounts for seasonality and external factors.


\section{Methodology}
\subsection{Data}

This work addresses forecasting the number of sales for a retail chain in Ecuador for 2016. The \href{https://www.kaggle.com/c/store-sales-time-series-forecasting/data}{data} were taken from Kaggle. Three files were used (train.csv, stores.csv, oil.csv), from which the final dataset was assembled.

Description of the variables in train.csv:
\begin{itemize}
	\item $store\_nbr$ identifies the store where the goods are sold.
	\item $family$ identifies the type of product sold.
	\item $sales$ is the total sales for a given product group in a particular store on a given date. Fractional values are possible, since goods can be sold in fractional units (e.g., 1.5 kg of cheese rather than 1 bag of chips).
	\item $onpromotion$ is the total number of items in a product group that were on promotion in the store on a given date.
\end{itemize}

The stores.csv file contains store metadata, including city, state, type, and cluster (all stores were grouped by similarity). The oil.csv file contains daily oil prices, covering both the training and the test periods. (Ecuador is an oil-dependent country, and its economic condition is strongly affected by oil-price fluctuations.)

\subsection{Models and their validation}

Three models were considered for forecasting the time series:
\begin{itemize}
	\item ARIMA model with manual tuning of the parameters p, d, and q
	\item SARIMAX model
	\item Machine-learning model (gradient boosting, XGBRegressor)
\end{itemize}


Quality was assessed using the following metrics: MSE, RMSE, MAE, and absolute percentage error. The results were compared against a naive forecast --- the average number of sales over the last 3 months.

For the SARIMAX and boosting models, two approaches were considered. In the first case, the model was fed time-based features: the index of the month, quarter, day of the year, and week, as well as the day of the week. In addition, ``lag features'' were used: the number of sales on the same date one, two, and three years earlier (where data for that period exist). This approach allows forecasting any number of days ahead. In the second case, dataset-derived variables were added to these features: the fuel price on the chosen date, the number of promotions in stores, and the number of transactions. The difficulty of the second approach is that we do not know the future values of the added drivers in advance. To forecast dates not present in our dataset, the driver values must also be predicted.

\subsubsection{ARIMA model}

The parameters for the ARIMA model were selected based on the autocorrelation and partial-autocorrelation functions. Stationarity of the series was tested using the Dickey--Fuller test.

It was found that the original series is not stationary ($p\text{-value}: 0.68$). After a single differencing, the series became stationary ($p\text{-value}: 6.33 \cdot 10^{-14}$). In addition, a distribution plot of the residuals was produced; it turned out to be close to normal.

The results of the ARIMA model are shown in Figure \ref{fig:arima}.

\subsubsection{SARIMAX model}

The main difference of the SARIMAX model from the ARIMA model is its ability to account for seasonality in the prediction, as well as additional parameters beyond the values of the series itself. As mentioned above, the additional parameters fed to the model were the number of sales on the date, the fuel price, and the number of promotions in stores. The model was trained automatically using the Python $statsmodels$ package. The model was trained twice: first on time and lag features only, and second with the additional variables.
The model predictions are shown in Figure \ref{fig:sarimax}. The metric values are presented in Table \ref{tab:sarimax}.

As an extension to the work, it was of interest to see how boosting performs on the same input data. The working principle of boosting is fundamentally different from the econometric models considered above.

The work used XGBRegressor from the $sklearn$ library. As with the SARIMAX model, training was carried out using two approaches. Hyperparameter search was performed with the $hyperopt$ library. Unlike an ordinary grid search, this approach uses Bayesian optimization. In addition, the hyperparameter search used cross-validation adapted to time series.

\section{Conclusion}

This work solved the problem of forecasting the number of sales for a retail chain in Ecuador for 2016. Three forecasting approaches were considered:
\begin{itemize}
	\item ARIMA model with manual tuning of the parameters p, d, and q
	\item SARIMAX model
	\item Machine-learning model (gradient boosting, XGBRegressor)
\end{itemize}

It was found that, among the econometric models, SARIMAX performed best. This result is due in particular to its ability to account for external parameters, rather than only the past values of the series.

In addition, the work considered forecasting with gradient boosting. This method outperformed all econometric models in prediction quality.
